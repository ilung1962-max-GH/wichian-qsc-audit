# ระบบ Smart QSC Audit & Franchise Standard System 🥟✨
### สำหรับระบบร้านอาหารแฟรนไชส์ (กรณีศึกษา: วิเชียรซาลาเปา · ขนมจีบ)

แอปพลิเคชันบนเว็บแบบ Responsive เต็มรูปแบบ (Mobile, Tablet, Desktop) ออกแบบมาสำหรับการเข้าประเมินมาตรฐานคุณภาพ บริการ และความสะอาด (QSC) ตามข้อกำหนดและเกณฑ์ประเมินของ **กรมพัฒนาธุรกิจการค้า (DBD) พ.ศ. 2569 หมวด 6 การปฏิบัติการ** ที่กำหนดให้ผู้รับสิทธิ์แฟรนไชส์ต้องได้รับการตรวจสอบมาตรฐานอย่างน้อยปีละ 1 ครั้ง

---

## 🌟 คุณสมบัติเด่นของระบบ (Core Features)

1. **สลับบทบาทเดโมในตัวแอป (Demo Role Switcher):** ทดสอบสลับมุมมองการทำงานได้ 3 บทบาทจากมุมขวาบนของระบบ:
   - 👑 **ผู้จัดการแฟรนไชส์ (HQ):** ดูแลระบบโดยรวม ทะเบียนร้าน ผู้ตรวจ วางแผนงาน และดูแดชบอร์ดสรุปคะแนน
   - 🔍 **ผู้ตรวจประเมิน (Auditor):** รับมอบหมายตารางเวลา ลงพื้นที่บันทึกผลการตรวจแนบรูปถ่าย ออกรายงานเป็น PDF และทำรายงานติดตามผล
   - 🏪 **แฟรนไชส์ซี (เจ้าของสาขา):** ดูเกรดคะแนนสะสม และรายงานการแก้ไขปัญหา RCA 3 ส่วน
2. **การทำงานแบบออฟไลน์ (Offline-first):** ระบบบันทึกร่างและประเมินผลการตรวจลงในฐานข้อมูลเครื่อง (**IndexedDB** ผ่าน **Dexie.js**) โดยไม่สูญหายแม้ไม่มีสัญญาณอินเทอร์เน็ตหน้างาน
3. **การประเมินสดคำนวณสด (Weighted Live Scoring):** ระบบคิดคะแนนและเกรดเฉลี่ย (A, B, C, D) แบบถ่วงน้ำหนักรายหมวดหมู่แบบสดๆ ขณะกรอกข้อมูล
4. **ความปลอดภัยทางธุรกิจและมาตรฐานสูงสุด (Business Rules Enforced):**
   - **Training Gate:** มอบหมายแผนงานให้เฉพาะผู้ตรวจที่สอบผ่านเกณฑ์อบรม และใบรับรอง Cert ยังไม่หมดอายุ
   - **Conflict of Interest Check:** ระบบจะบล็อกหรือแจ้งเตือนเมื่อมอบหมายผู้ตรวจที่มีผลประโยชน์ชนกับร้านสาขานั้นๆ
   - **Annual Coverage Warning:** แสดงกล่องแจ้งเตือนสีแดงเชิงรุกหากมีสาขาแฟรนไชส์ซีที่ยังไม่มีกำหนดแผนการตรวจในปีนี้
   - **Mystery Confidentiality:** สาขาจะไม่เห็นตารางงานประเมินประเภท Mystery ล่วงหน้า เพื่อป้องกันการจัดเตรียมความพร้อม
   - **Different Follow-up Auditor:** การติดตามผลความคืบหน้า (Verify CAPA) ต้องใช้นายตรวจคนใหม่ ห้ามใช้นายตรวจคนเดิมเพื่อความโปร่งใส

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

- **Frontend:** React 18 + TypeScript + Vite
- **Styling:** Tailwind CSS (Responsive สมบูรณ์แบบ)
- **State Management:** Zustand (จัดการ Reactive State)
- **Local Storage:** Dexie.js (IndexedDB Wrapper สำหรับออฟไลน์)
- **Chart:** Recharts (แดชบอร์ดสรุปยอด)
- **Icons:** Lucide React

## 🚀 วิธีการทดสอบและเปิดใช้งานระบบ (Getting Started)

### 1. ติดตั้งไลบรารี
ตรวจสอบว่าเครื่องของคุณมี Node.js แล้ว จากนั้นเปิด PowerShell หรือ Terminal ในโฟลเดอร์นี้แล้วพิมพ์:
```bash
npm install
```

### 2. รันแอปพลิเคชันในโหมดพัฒนา (Development Mode)
```bash
npm run dev
```
ระบบจะเปิดพอร์ตการทำงาน เช่น `http://localhost:5173/` คุณสามารถคัดลอกลิงก์ไปเปิดบนเว็บบราวเซอร์ เช่น Google Chrome เพื่อเริ่มเล่นและตรวจสอบการตอบสนองหน้าจอได้ทันที

### 3. รันโปรเจกต์แบบพร้อมใช้งานจริง (Build & Preview)
```bash
npm run build
npm run preview
```

## 🔍 แผนพัฒนาในระยะถัดไป (Phase 2 Roadmap)

หากระบบต้องการนำไปใช้ปฏิบัติงานจริงกับสาขาทั่วประเทศ จะขยายการติดตั้งระบบ Backend ดังนี้:
1. **คลาวด์เซิร์ฟเวอร์ฐานข้อมูล:** เชื่อมต่อ API และระบบจัดการสิทธิ์เข้าใช้งานผ่าน **Supabase** (PostgreSQL)
2. **ระบบการยืนยันตัวตน (Authentication):** แบ่งแยกอีเมลผู้ใช้งานและรหัสผ่านจริงตามบทบาท (HQ, นายตรวจ, เจ้าของร้าน)
3. **จัดเก็บรูปถ่ายความเสียหายจริง (Object Storage):** อัปโหลดภาพ Before/After จากสมาร์ทโฟนของนายตรวจและร้านขึ้นไปเก็บบนระบบ Supabase Storage หรือ AWS S3
4. **ระบบแจ้งเตือนเชิงรุก (Notifications):** ส่งพุชแจ้งเตือนใบงานด่วน หรือส่งรายงานผลตรวจ PDF เข้าทางอีเมลและแชทไลน์ของเจ้าของสาขาโดยตรง
